## 一、接口说明

### 1.1 功能描述

获取KOL在指定时间范围内的数据概览，包括流量数据、互动数据、转化数据等核心运营指标。

### 1.2 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `kolId` | string | 是 | KOL的星图ID | "6831825876674412552" |
| `_type` | string | 是 | 视频类型<br>• `_1`: 个人视频(personal video)<br>• `_2`: 星图视频(xingtu video) | "_1" |
| `_range` | string | 是 | 时间范围<br>• `_2`: 近30天(last 30 days)<br>• `_3`: 近90天(last 90 days) | "_2" |
| `flowType` | int | 是 | 流量类型，默认为1 | 1 |

## 三、核心数据字段 (data)

### 3.1 基础响应字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `base_resp` | object | 基础响应信息 |
| `base_resp.status_code` | int | 状态码，0表示成功 |
| `base_resp.status_message` | string | 状态消息 |

### 3.2 内容数据指标

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `avg_duration` | string | 平均视频时长 | "21319" (秒) |
| `item_num` | string | 统计期内发布的作品数量 | "1" |

**说明**:
- `avg_duration`: 单位为秒，"21319"表示约21秒
- `item_num`: 在指定时间范围内发布的视频数量

### 3.3 互动数据指标

| 字段名 | 类型 | 说明 | 单位 |
|--------|------|------|------|
| `like_avg` | string | 平均点赞数 | 次 |
| `comment_avg` | string | 平均评论数 | 次 |
| `share_avg` | string | 平均分享数 | 次 |

**示例**:
```json
{
  "like_avg": "8291",
  "comment_avg": "202",
  "share_avg": "79"
}
```

### 3.4 播放数据指标

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `play_mid` | string | 播放量中位数 |

**示例**: 
```json
{
  "play_mid": "420118"
}
```

**说明**: 
- 中位数比平均值更能反映真实的播放水平，不受个别爆款视频影响

### 3.5 互动率 (interact_rate)

**数据结构**:
```json
{
  "label": "美妆",
  "overtake": 6237,
  "value": 204
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `label` | string | 所属行业标签 |
| `overtake` | int | 超过的同行业KOL数量（千分位） |
| `value` | int | 互动率数值（千分位，需÷100得到百分比） |

**示例计算**:
- `value: 204` → 实际互动率 = 204 ÷ 10000 = 2.04%
- `overtake: 6237` → 超过62.37%的美妆行业KOL

**业务含义**:
- **互动率** = (点赞数 + 评论数 + 分享数) / 播放量
- 反映内容的吸引力和用户参与度
- 行业对比帮助评估KOL的相对质量

### 3.6 完播率 (play_over_rate)

**数据结构**:
```json
{
  "label": "美妆",
  "overtake": 2012,
  "value": 17
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `label` | string | 所属行业标签 |
| `overtake` | int | 超过的同行业KOL数量（千分位） |
| `value` | int | 完播率数值（千分位，需÷100得到百分比） |

**示例计算**:
- `value: 17` → 实际完播率 = 17 ÷ 10000 = 0.17%
- `overtake: 2012` → 超过20.12%的美妆行业KOL

**业务含义**:
- **完播率** = 完整观看视频的人数 / 总播放量
- 反映内容质量和用户粘性
- 完播率越高，说明内容越吸引人

### 3.7 播放稳定性 (play_stability)

**数据结构**:
```json
{
  "label": "美妆",
  "overtake": 9083,
  "value": 528863
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `label` | string | 所属行业标签 |
| `overtake` | int | 超过的同行业KOL数量（千分位） |
| `value` | int | 稳定性数值 |

**业务含义**:
- 反映KOL播放量的稳定性和可预测性
- 数值越高，说明每个视频的播放量越稳定
- 对于广告主来说，稳定性高的KOL风险更低

### 3.8 作品相关指标 (item_rate)

**数据结构**:
```json
{
  "interact_rate": {},
  "item_num": {
    "label": "",
    "overtake": 6700,
    "value": 1
  },
  "play_mid": {
    "label": "",
    "overtake": 9957,
    "value": 420118
  }
}
```

| 字段名 | 说明 |
|--------|------|
| `item_num` | 作品数量的行业对比 |
| `play_mid` | 播放量中位数的行业对比 |
| `interact_rate` | 互动率的行业对比（可能为空） |

**业务含义**:
- 对核心指标进行行业横向对比
- `overtake`值越高，说明在同行业中表现越好

## 四、商业报价指标

### 4.1 预期CPE (expect_cpe)

**Cost Per Engagement - 每次互动成本**

**数据结构**:
```json
{
  "cpe_1_20": "252",
  "cpe_21_60": "396",
  "cpe_60": "505"
}
```

| 字段名 | 说明 | 单位 |
|--------|------|------|
| `cpe_1_20` | 1-20秒视频的每次互动成本 | 分 |
| `cpe_21_60` | 21-60秒视频的每次互动成本 | 分 |
| `cpe_60` | 60秒以上视频的每次互动成本 | 分 |

**示例计算**:
```
cpe_1_20: "252" = 2.52元/次互动
cpe_21_60: "396" = 3.96元/次互动
cpe_60: "505" = 5.05元/次互动
```

**业务含义**:
- CPE = 广告投放价格 / 预期互动次数
- CPE越低，性价比越高
- 适用于追求用户参与度的营销目标

### 4.2 预期CPM (expect_cpm)

**Cost Per Mille - 每千次播放成本**

**数据结构**:
```json
{
  "cpm_1_20": "7015",
  "cpm_21_60": "11025",
  "cpm_60": "14031"
}
```

| 字段名 | 说明 | 单位 |
|--------|------|------|
| `cpm_1_20` | 1-20秒视频的每千次播放成本 | 分 |
| `cpm_21_60` | 21-60秒视频的每千次播放成本 | 分 |
| `cpm_60` | 60秒以上视频的每千次播放成本 | 分 |

**示例计算**:
```
cpm_1_20: "7015" = 70.15元/千次播放
cpm_21_60: "11025" = 110.25元/千次播放
cpm_60: "14031" = 140.31元/千次播放
```

**业务含义**:
- CPM = 广告投放价格 / (预期播放量 / 1000)
- CPM越低，曝光成本越低
- 适用于追求品牌曝光的营销目标

### 4.3 销售预期CPM (sales_expect_cpm)

**数据结构**:
```json
{
  "cpm_1_20": "7229",
  "cpm_21_60": "11359",
  "cpm_60": "14457"
}
```

| 字段名 | 说明 |
|--------|------|
| `cpm_1_20` | 带货场景下的1-20秒视频CPM |
| `cpm_21_60` | 带货场景下的21-60秒视频CPM |
| `cpm_60` | 带货场景下的60秒以上视频CPM |

**说明**:
- 针对电商带货场景的CPM预估
- 通常比普通CPM略高，因为考虑了转化因素
- 单位同样为"分"，需÷100转为元

## 五、使用建议

### 5.1 关键指标选择

**评估KOL质量**:
1. ✅ `interact_rate.value` > 200 (互动率 > 2%)
2. ✅ `play_over_rate.value` > 50 (完播率 > 0.5%)
3. ✅ `play_stability.overtake` > 5000 (播放稳定性超过50%的同行)

**评估性价比**:
1. ✅ `expect_cpm.cpm_21_60` < 20000 (CPM < 200元)
2. ✅ `expect_cpe.cpe_21_60` < 1000 (CPE < 10元)
3. ✅ 对比 `overtake` 值，选择行业头部KOL

**评估投放风险**:
1. ✅ `play_stability.value` 越高越稳定
2. ✅ `item_num` ≥ 3 (有足够的内容产出)
3. ✅ 查看 `play_mid` 而非平均值（避免爆款误导）

### 5.2 价格计算示例

**场景**: 投放21-60秒视频广告

```python
# 原始数据
cpm_21_60 = 11025  # 单位：分
cpe_21_60 = 396    # 单位：分
play_mid = 420118  # 播放量中位数
interact_rate_value = 204  # 互动率（千分位）

# 转换为实际值
cpm = cpm_21_60 / 100  # 110.25元/千次播放
cpe = cpe_21_60 / 100  # 3.96元/次互动
actual_interact_rate = interact_rate_value / 10000  # 2.04%

# 计算投放效果
expected_play = play_mid  # 预期播放量
expected_interact = expected_play * actual_interact_rate  # 预期互动次数

# 反推广告价格
ad_price_by_cpm = (expected_play / 1000) * cpm
ad_price_by_cpe = expected_interact * cpe

print(f"预期播放: {expected_play:,}次")
print(f"预期互动: {expected_interact:,.0f}次")
print(f"按CPM估算价格: {ad_price_by_cpm:,.2f}元")
print(f"按CPE估算价格: {ad_price_by_cpe:,.2f}元")
```

### 5.3 注意事项

1. **数值单位转换**:
   - 价格字段（CPE、CPM）单位为"分"，需÷100转为元
   - 互动率、完播率等百分比字段使用千分位表示，需÷10000转为小数

2. **行业对比**:
   - `label` 字段标识所属行业
   - `overtake` 表示超过的同行业KOL百分比（千分位）
   - 不同行业的数值不具备横向可比性

3. **时间范围影响**:
   - `_range=_2` (30天) 数据更实时，但样本量较小
   - `_range=_3` (90天) 数据更稳定，但可能不够新

4. **视频类型区分**:
   - `_type=_1` (个人视频): 包含所有视频
   - `_type=_2` (星图视频): 仅包含商业合作视频
   - 个人视频数据更能反映真实实力

5. **空值处理**:
   - 部分KOL可能某些字段为空对象 `{}`
   - 说明该KOL在该指标上缺乏数据或不符合统计条件
   - 需要在代码中做好空值判断

## 六、数据质量评估

### 6.1 数据完整性检查

必需字段检查：
- [ ] `base_resp.status_code` = 0
- [ ] `play_mid` 存在且 > 0
- [ ] `interact_rate` 存在且不为空对象

### 6.2 数据有效性判断

有效KOL标准：
- [ ] `play_mid` > 10000 (播放量 > 1万)
- [ ] `interact_rate.value` > 100 (互动率 > 1%)
- [ ] `item_num` > 0 (有内容产出)
- [ ] `expect_cpm` 不为空（有报价）

### 6.3 异常数据处理

可能的异常情况：
1. **所有字段都是空对象**: KOL数据不足或账号异常
2. **overtake值为0**: 在行业中排名靠后
3. **CPM/CPE极高**: 可能是头部KOL或数据异常
4. **item_num为0**: 统计期内无内容产出
